Index: General (for Resume Operations)

Name: Name of the index.

Schema: Schema that owns the index.

Index On:

  • Schema: Schema that owns the table containing the column on which this index is created.
  • Table: Table containing the column on which this index is created.
  • Text Column: Column on which this index is created. This column contains the text of the document (row) to be processed. Can be a single-column text key or an encoded specification for a composite (multiple-column) text key.

Resume: Causes the CREATE INDEX operation to be resumed.

Memory:

  • Use Default Memory (12 M): Use the default amount of memory for indexing (if creating the index) or for the synchronize operation (if synchronizing the index).
  • Set Memory Use: Amount of run-time memory to use for or indexing (if creating the index) or for the synchronize operation (if synchronizing the index). Specify a number and the unit of measurement (Megabytes or Gigabytes).

Set Memory Use must be between 1 megabyte and the value specified for max_index_memory in CTX_PARAMETERS. The default is the value specified for default_index_memory in CTX_PARAMETERS. If you are editing an index, memory size is the amount of memory used for the ALTER INDEX operation before flushing the index to disk.

Specifying a large amount memory improves indexing performance because there is less I/O, and it improves query performance and maintenance because there is less fragmentation. Specifying smaller amounts of memory increases disk I/O and index fragmentation, but might be useful if you want to track indexing progress or when runtime memory is scarce.

Note: If you encounter any error or warning messages, you can view the details by starting SQL*Plus Worksheet and entering the following statement: select * from dr$index_error;

Note: To perform any of the following operations on an index, you must use the Object menu or the right-mouse menu:

  • Resume an index creation operation that had been stopped
  • Synchronize the index to reflect rows that have been updated, inserted, and deleted in the table
  • Optimize the index (compact fragmented rows in the index and, for full optimization, remove old data)
